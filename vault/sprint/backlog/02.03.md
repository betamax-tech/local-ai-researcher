## Objective
Implement error hierarchy, codes, and boundaries so all operations fail with predictable, recoverable exceptions.

## Acceptance Criteria
- [ ] `src/errors/index.ts` exports error classes: `ResearcherError`, `SearchError`, `ReadError`, `GatherError`, `HealthError`.
- [ ] All error codes from PRD taxonomy (`SEARCH_TIMEOUT`, `READ_NOT_FOUND`, etc.) are concrete `const` values.
- [ ] Each error has: code, message, retriable flag, data (optional).
- [ ] Errors can be serialized to JSON-RPC error format.
- [ ] Unit tests verify all error codes are caught and not leaked as `Error`.

## Architecture Notes

**Error hierarchy:**
```
ResearcherError (base)
├── SearchError
├── ReadError
├── GatherError
└── HealthError
```

Retriable errors: `SEARCH_TIMEOUT`, `SEARCH_SOURCE_UNAVAILABLE`, `GATHER_TIMEOUT`, `GATHER_PARTIAL`, `HEALTH_CHECK_FAILED`.
