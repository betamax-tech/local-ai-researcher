## Objective
Implement request-scoped deduplication so that within a single `gather()` call, duplicate tool invocations are eliminated.

## Acceptance Criteria
- [ ] `src/dedup/engine.ts` exports `DedupEngine` class.
- [ ] Each `gather()` call creates a new dedup context (request-scoped, not global).
- [ ] `canHandle(source, target)` checks if URL or file has been read in current request.
- [ ] `recordResult(source, target, result)` stores result and marks as deduplicated.
- [ ] Dedup stats returned with `GatherResult`: `{ total: 5, deduped: 2 }`.
- [ ] URL canonicalization rules from PRD applied consistently.
- [ ] Unit tests verify: same URL twice → deduped; different params → separate requests.

## Architecture Notes

**Dedup strategy:**
1. Normalize URLs (canonicalization rules from PRD).
2. Compute deterministic hash of `source + target`.
3. Check if hash exists in current request context.
4. If yes, return cached result; if no, execute and cache.

Dedup is NOT global (no TTL, no cross-request cache). It's purely within-request.
