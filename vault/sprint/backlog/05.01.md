## Objective
Implement the CLI entrypoint with config loading, argument parsing, and structured logging.

## Acceptance Criteria
- [ ] `src/cli.ts` exports async `main(argv)` function.
- [ ] Command-line args: `--config`, `--log-level`, `--cwd`, `--api-key`, etc.
- [ ] Config loading order: environment vars → `~/.local-researcher/config.json` → CLI args.
- [ ] Supports JSON and YAML config files.
- [ ] Initializes logger based on `LOG_LEVEL` env var (default `info`).
- [ ] Registers all tools with MCP server on startup.
- [ ] CLI commands:
  - [ ] `health`: check server status
  - [ ] `search --query "..." --sources web,local`: run search
  - [ ] `read --path "file.ts"`: read file
  - [ ] `gather --prompt "..." --sources web,local`: orchestrate gather
  - [ ] `serve`: run MCP server continuously
- [ ] Help text for all commands.
- [ ] Graceful shutdown (Ctrl+C).
- [ ] Exit codes: 0 (success), 1 (error), 2 (invalid args).
- [ ] Integration test: CLI startup → health check → exit.

## Architecture Notes

**Config resolution:** Environment overrides config file, CLI args override both.

**Logger:** Structured JSON output (each log entry is a JSON object on one line).
