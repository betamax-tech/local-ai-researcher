## Objective
Implement the `gather()` function to orchestrate `search()` and `read()` across multiple sources, apply dedup, and synthesize results into an LLM-ready context block.

## Acceptance Criteria
- [ ] `src/tools/gather.ts` exports async `gather(prompt, sources, options)` function.
- [ ] Creates new dedup context for this request.
- [ ] Processes sources in parallel (default) or sequential (option).
- [ ] Calls `search()` or `read()` based on source type.
- [ ] Collects results and applies dedup.
- [ ] Synthesizes results into formatted context:
  ```
  # Research Context for: [prompt]

  **Sources Queried:** [list]
  **Dedup Stats:** [total X, deduped Y]

  ## Source 1: [title] ([url])
  [excerpt or content]

  ## Source 2: ...
  ```
- [ ] Returns `GatherResult` with unique `id`, synthesis, and dedup stats.
- [ ] Timeout enforced (default 10s; configurable).
- [ ] Respects `maxTokens` option for sampling (approximately).
- [ ] Throws `GatherError` with codes: `GATHER_NO_SOURCES`, `GATHER_TIMEOUT`, `GATHER_PARTIAL`.
- [ ] Unit tests: single source, multi-source parallel, dedup verification, timeout, partial failure.

## Architecture Notes

**Synthesis format:** Markdown, LLM-friendly, includes source metadata.

**Request ID:** Unique per `gather()` call, used for logging and audit trail.
