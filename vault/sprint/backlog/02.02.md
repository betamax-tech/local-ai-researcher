## Objective
Define TypeScript interfaces for all canonical tools: `search`, `read`, `gather`, and `health`.

## Acceptance Criteria
- [ ] `src/types/index.ts` exports all types from PRD.
- [ ] `SearchOptions`, `SearchResult`, `ReadOptions`, `ReadResult`, `GatherOptions`, `GatherResult`, `HealthResult` all defined.
- [ ] All interfaces use strict TypeScript (no `any`).
- [ ] Error types enum includes all codes from PRD (SEARCH_TIMEOUT, READ_NOT_FOUND, etc.).
- [ ] Tool input/output schemas can be serialized to JSON Schema for MCP registration.
- [ ] All types pass `npm run type-check`.

## Architecture Notes

**Organization:**
```
src/types/
  index.ts (exports)
  search.ts
  read.ts
  gather.ts
  health.ts
  errors.ts
```
