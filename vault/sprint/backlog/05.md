## Objective
Implement the `health()` tool to report MCP server readiness, source availability, and resource usage.

## Acceptance Criteria
- [ ] `src/tools/health.ts` exports async `healthTool()` function.
- [ ] Returns `HealthResult` with status: `healthy`, `degraded`, or `unhealthy`.
- [ ] MCP stdio transport status: ready bool, version string.
- [ ] Registered sources: array of `{ name, status: 'connected'|'error', error?: string }`.
- [ ] Resource stats: memory (MB), current working directory.
- [ ] Checks:
  - [ ] Stdio transport is listening.
  - [ ] Each source adapter is reachable (web API, local filesystem, custom MCP).
  - [ ] Memory usage < 500MB (warn if > 250MB).
- [ ] Status logic:
  - [ ] `healthy`: all sources connected, memory OK.
  - [ ] `degraded`: 1+ source unavailable, but core functionality works.
  - [ ] `unhealthy`: MCP transport down or memory critical.
- [ ] Unit tests: healthy case, missing API key, MCP down, high memory.

## Architecture Notes

**Health check is lightweight:** Quick connectivity check, not full tool invocation.
