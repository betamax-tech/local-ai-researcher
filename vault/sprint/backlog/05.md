## Objective
Implement `health()` tool to report MCP server readiness, provider availability, and resource usage.

## Acceptance Criteria
- [x] `src/tools/health.ts` exports `createHealthTool()` function.
- [x] Returns `HealthResult` with status: `healthy`, `degraded`, or `unhealthy`.
- [x] MCP stdio transport status: ready bool, version string.
- [x] Provider health: array of `{ name, status: 'connected'|'error', error?: string }`.
- [x] Resource stats: memory (MB), current working directory.
- [x] Checks:
  - [x] Stdio transport is listening (via MCP SDK).
  - [x] Each provider is reachable (SearxNG, Jina Reader).
  - [x] Memory usage reported (no hard limit in v1).
- [x] Status logic:
  - [x] `healthy`: all providers connected.
  - [x] `degraded`: 1+ provider unavailable, but core functionality works.
  - [x] `unhealthy`: no providers connected.
- [x] ISO timestamp included in result.

## Architecture Notes

**Health check implementation:**
- Health tool in `src/tools/health.ts` calls both providers' `isHealthy()` methods
- Health check is lightweight (connectivity check, not full tool invocation)
- Timeout for health checks: 5000ms (hardcoded in providers)
- Health status based on provider connectivity

**Health result structure:**
```typescript
{
  status: 'healthy' | 'degraded' | 'unhealthy',
  mcp: {
    stdio: { ready: true, version: '1.0.0' },
    servers: [
      { name: 'SearxNG', status: 'connected' | 'error', error?: string },
      { name: 'Jina Reader', status: 'connected' | 'error', error?: string }
    ]
  },
  resources: {
    memoryMB: number,
    cwd: string
  },
  timestamp: string
}
```

**Implementation location:**
- `src/tools/health.ts`: Full health tool implementation
