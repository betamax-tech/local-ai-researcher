## Objective
Implement structured JSON logging throughout the codebase, audit trail recording, and complete v1 API documentation.

## Acceptance Criteria

### Logging
- [ ] All tool calls logged with: timestamp, method, params (sanitized), result/error.
- [ ] Request ID propagated through all calls.
- [ ] Structured JSON format: `{ timestamp, level, requestId, method, duration, status, ... }`.
- [ ] Log levels: `debug`, `info`, `warn`, `error`.
- [ ] Logs written to stdout (JSON lines format).
- [ ] Optional file logging: `LOCAL_RESEARCHER_LOG_FILE=/path/to/logs.jsonl`.
- [ ] No sensitive data logged (API keys redacted).

### Audit Trail
- [ ] Each `gather()` call produces an audit record: `{ id, prompt, sources, results, dedup, timestamp }`.
- [ ] Audit records can be written to file or database (config option).
- [ ] Audit trail enables reproducibility: re-run same `gather()` request ID reproduces same results.

### Documentation
- [ ] `README.md` with:
  - [ ] Installation (npm install)
  - [ ] Quick start (CLI example)
  - [ ] Configuration (env vars, config file)
  - [ ] Supported sources (web, local, custom)
  - [ ] Error codes and troubleshooting
- [ ] `API.md` with:
  - [ ] Function signatures (all public exports)
  - [ ] Type definitions (all interfaces)
  - [ ] Examples (search, read, gather, health)
  - [ ] Error handling guide
- [ ] Inline JSDoc comments on all public functions.
- [ ] CONTRIBUTING.md (development setup, test running, PR guidelines).

## Architecture Notes

**Logger:** Export singleton logger instance; all modules import and use.

**Audit trail:** Optional (opt-in via config). Default: disabled to reduce I/O.
