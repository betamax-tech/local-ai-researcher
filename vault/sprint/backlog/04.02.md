## Objective
Implement a registry for custom MCP servers and a generic adapter to invoke tools on external MCP servers.

## Acceptance Criteria
- [ ] `src/sources/mcp.ts` exports `MCPServerAdapter` class.
- [ ] Registry stores configured MCP server endpoints (names, stdio paths, capabilities).
- [ ] `search()` with `sources: ['custom']` dispatches to registered MCP servers.
- [ ] Calls remote MCP tool (e.g., `web_search` on custom server) via stdio transport.
- [ ] Maps custom tool results to canonical `SearchResult` format.
- [ ] Handles tool not found, timeout, and other MCP errors gracefully.
- [ ] Throws `SearchError` if MCP server unavailable.
- [ ] Config loading from `config.json` or environment variables.
- [ ] Unit tests: successful MCP call, tool not found, timeout.

## Architecture Notes

**Registry:** Simple map of server name → server config (path, timeout, tool mappings).

**Adapter:** Reuses `MCPServer` transport to call tools on remote servers.
