## Objective
Verify that provider health checks work correctly for both SearxNG and Jina Reader.

## Acceptance Criteria
- [ ] SearxNG health check returns `true` when endpoint is reachable.
- [ ] SearxNG health check returns `false` when endpoint is unreachable or times out.
- [ ] Jina Reader health check returns `true` when endpoint is reachable.
- [ ] Jina Reader health check returns `false` when endpoint is unreachable or times out.
- [ ] Health check responses logged appropriately.
- [ ] Health tool returns correct status: 'healthy', 'degraded', or 'unhealthy'.
- [ ] Overall health status calculation verified:
  - [ ] All providers connected → 'healthy'
  - [ ] Some providers connected → 'degraded'
  - [ ] No providers connected → 'unhealthy'
- [ ] Unit tests for health check logic.
- [ ] Integration tests with mock and real endpoints.

## Architecture Notes

**Health check implementation:**
- Both providers implement `isHealthy()` async method
- Health tool in `src/tools/health.ts` calls both providers
- Health check is lightweight (not full tool invocation)
- Timeout for health checks: 5000ms
- Health status based on provider connectivity

**Health result structure:**
```typescript
{
  status: 'healthy' | 'degraded' | 'unhealthy',
  mcp: {
    stdio: { ready: true, version: '1.0.0' },
    servers: [
      { name: 'SearxNG', status: 'connected' | 'error', error?: string },
      { name: 'Jina Reader', status: 'connected' | 'error', error?: string }
    ]
  },
  resources: {
    memoryMB: number,
    cwd: string
  },
  timestamp: string
}
```

**Implementation locations:**
- `src/providers/searxng.ts`: `isHealthy()` method (line ~104)
- `src/providers/jinaReader.ts`: `isHealthy()` method (line ~88)
- `src/tools/health.ts`: Health tool implementation
