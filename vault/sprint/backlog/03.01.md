## Objective
Implement the `read()` tool for safe, sandboxed file access with line-range support and encoding detection.

## Acceptance Criteria
- [ ] `src/tools/read.ts` exports async `readTool(path, options)` function.
- [ ] Resolves file paths relative to `cwd` (from env/config).
- [ ] Respects `.gitignore` (do not read ignored files).
- [ ] Supports line range: `read(path, { lines: [10, 50] })` extracts lines 10–50 (1-indexed).
- [ ] Full file read if no `lines` option.
- [ ] Handles `utf-8` and `binary` encodings (default `utf-8`).
- [ ] Returns `ReadResult` with `path`, `content`, `encoding`, `lineCount`.
- [ ] Throws `ReadError` with codes: `READ_NOT_FOUND`, `READ_PERMISSION_DENIED`, `READ_ENCODING_ERROR`.
- [ ] Unit tests: basic read, line range, missing file, permission denied, encoding error.

## Architecture Notes

**Sandboxing:** All reads are relative to `process.cwd()` (or `LOCAL_RESEARCHER_CWD` env var). Resolve paths with `path.resolve(cwd, userPath)` and verify result is within `cwd` (no `../` escapes).

**Dedup integration:** `read()` results are deduped by path + line range within a `gather()` call.
