## Objective
Verify that provider error handling follows the locked v1 error taxonomy and propagates errors correctly.

## Acceptance Criteria
- [ ] SearxNG errors mapped correctly:
  - [ ] Timeout → `ERR_SEARXNG_TIMEOUT` (retryable: true)
  - [ ] Unavailable → `ERR_SEARXNG_UNAVAILABLE` (retryable: true)
  - [ ] Invalid response → `ERR_SEARXNG_INVALID_RESPONSE` (retryable: false)
- [ ] Jina Reader errors mapped correctly:
  - [ ] Timeout → `ERR_JINA_TIMEOUT` (retryable: true)
  - [ ] Unavailable → `ERR_JINA_UNAVAILABLE` (retryable: true)
  - [ ] Invalid response → `ERR_JINA_INVALID_RESPONSE` (retryable: false)
- [ ] Tool errors wrapped in proper error envelopes:
  - [ ] `schema_version: '1'`
  - [ ] `ok: false`
  - [ ] `error.code` matches taxonomy
  - [ ] `error.message` is descriptive
  - [ ] `error.retryable` is correct
  - [ ] `error.details` includes context (when applicable)
- [ ] Error logging verified:
  - [ ] All errors logged with appropriate level (warn, error)
  - [ ] Error details included in logs
  - [ ] No sensitive information leaked in logs
- [ ] Unit tests for error scenarios.
- [ ] Integration tests with failing endpoints.

## Architecture Notes

**Error taxonomy (locked v1):**
```typescript
// SearxNG errors
ERR_SEARXNG_TIMEOUT (retryable: true)
ERR_SEARXNG_UNAVAILABLE (retryable: true)
ERR_SEARXNG_INVALID_RESPONSE (retryable: false)

// Jina Reader errors
ERR_JINA_TIMEOUT (retryable: true)
ERR_JINA_UNAVAILABLE (retryable: true)
ERR_JINA_INVALID_RESPONSE (retryable: false)

// Gather errors
ERR_GATHER_TIMEOUT (retryable: true)
ERR_GATHER_NO_SOURCES (retryable: false)

// Validation errors
ERR_VALIDATION (retryable: false)

// Generic errors
ERR_TIMEOUT (retryable: true)
ERR_UNAVAILABLE (retryable: true)
```

**Error envelope structure:**
```typescript
{
  schema_version: '1',
  ok: false,
  error: {
    code: 'ERR_SEARXNG_TIMEOUT',
    message: 'SearxNG search timed out',
    retryable: true,
    details: { query: 'test', duration: 10000 }
  }
}
```

**Implementation locations:**
- `src/lib/errors.ts`: Error class definitions
- `src/providers/searxng.ts`: SearxNG error handling (line ~202)
- `src/providers/jinaReader.ts`: Jina Reader error handling (line ~199)
- `src/tools/*.ts`: Tool error envelope wrapping
