## Objective
Implement the Model Context Protocol (MCP) `stdio` transport, including message framing, request/response handling, and error propagation.

## Acceptance Criteria
- [ ] `src/mcp/transport.ts` implements MCP JSON-RPC 2.0 message framing.
- [ ] `MCPServer` class can start listening on stdio and emit typed events for tool calls.
- [ ] Message validation passes schema checks (valid JSON-RPC structure).
- [ ] `initializeServer()` returns `ServerCapabilities` with tool list.
- [ ] Error handling: malformed messages logged, not crashed.
- [ ] Request timeouts enforced (default 10s).
- [ ] All MCP methods typed with `@types/mcp` or vendored types.

## Architecture Notes

**Key types:**
- `JSONRPCRequest`, `JSONRPCResponse` (JSON-RPC 2.0 spec)
- `Tool` (with name, description, inputSchema)
- `MCPServer` (event emitter for tool invocations)

Transport is request/response, not streaming in v1.
