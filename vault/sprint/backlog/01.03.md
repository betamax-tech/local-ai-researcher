## Objective
Set up ESLint with TypeScript rules and Prettier for consistent code formatting.

## Acceptance Criteria
- [ ] `.eslintrc.json` or `eslintrc.config.js` exists.
- [ ] ESLint includes `@typescript-eslint` parser and plugin.
- [ ] Rules enabled: `@typescript-eslint/strict`, `no-unused-vars`, `no-implicit-returns`, `prefer-const`.
- [ ] `.prettierrc.json` configured with project standards (2-space indent, 100 char line length).
- [ ] `npm run lint` passes on all seed TypeScript files.
- [ ] `npm run format` (Prettier) reformats all `src/**/*.ts` without errors.
- [ ] `.eslintignore` and `.prettierignore` include `node_modules/`, `dist/`.

## Architecture Notes

**Rules philosophy:** Strict TypeScript enforcement. No `any` unless explicitly justified in source comment.
